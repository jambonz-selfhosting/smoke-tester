# NNNN. Short decision title in imperative form

- **Status:** Proposed | Accepted | Deprecated | Superseded by ADR-NNNN
- **Date:** YYYY-MM-DD
- **Deciders:** names / roles
- **Tags:** optional, comma-separated (e.g. `sip`, `testing`, `tooling`)

## Context

What's the situation that forced this decision? What constraints, requirements, or prior decisions are in play? Keep it factual — no prescriptions yet.

## Decision

The decision itself, stated clearly and in active voice. One paragraph is usually enough. If there are sub-rules, use a bulleted list.

## Consequences

What becomes easier? What becomes harder? What follow-up work does this imply? Be honest about the downsides — future-you will thank you.

- Positive: …
- Negative: …
- Neutral / follow-up: …

## Alternatives considered

Each alternative gets a short paragraph naming it and saying *why it was rejected*. A decision without alternatives is not a decision — it is a default.

### Option A — <name>
Rejected because …

### Option B — <name>
Rejected because …

- Links to related ADRs, external docs, tickets, prior art.
